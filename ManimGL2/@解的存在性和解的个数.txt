# 游戏数学（一）点灯问题（Flip Game）的求解

## 1. 前言

点灯问题的规则很简单：点击一个格子，它自己以及上下左右相邻的格子都会翻转一次。因为同一个位置点两次等于没点，所以每个位置最多只需要点 0 次或 1 次。于是整个问题天然落在 F₂ 上，也就是“按或不按”“亮或灭”都可以看成 0/1 变量，所有运算都按模 2 进行。

最直接的算法，是枚举第一行或第一列的点击方式，共有 2ⁿ 种。之后每一行的点击方式，都会被上一行是否已经消掉唯一决定，因此可以一路递推到最后一行，再检查是否成立。这种方法虽然不是最优的，但已经能避免 O(2ⁿ²) 这种纯暴力。

不过这里我们更关心的不是算法，而是这个问题背后的数学结构：

1. 为什么解一定存在
2. 解有多少个
3. 为什么最后会落到 gcd(fₙ(x), fₙ(x+1)) 这样的形式

---

## 2. 解的存在性

### 2.1 一般图上的表述

先不局限于 n×n 棋盘，而考虑一个有 N 个顶点的无向图。每个顶点的状态只有两种：0 和 1。点击某个顶点时，它自己和所有邻点都会翻转，也就是加 1（模 2）。

设 A 为这个图的“点灯矩阵”，定义为：

Aᵢⱼ = 1，当且仅当 i=j 或 i,j 相邻
Aᵢⱼ = 0，其余情况

于是 A 是一个对称矩阵。若 x ∈ F₂ᴺ 表示每个点是否点击，b ∈ F₂ᴺ 表示目标变化量，那么问题就变成求解：

A x = b

特别地，如果我们希望把所有灯都翻转一次，那么就有：

b = (1,1,…,1)ᵀ

因此，问题“是否有解”，等价于“b 是否属于 Im A”。

---

### 2.2 用正交补判断是否有解

在线性代数里有一个标准结论：

(Ker Aᵀ)⊥ = Im A

所以只要证明：
对任意满足 Aᵀx=0 的向量 x，都有 bᵀx=0，
那么就能推出 b ∈ Im A，从而方程一定有解。

由于 A 是对称矩阵，所以 Aᵀ=A。对任意满足 Ax=0 的 x，有：

0 = xᵀAx
= ∑ᵢⱼ Aᵢⱼ xᵢxⱼ
= ∑ᵢ xᵢ² + 2∑ᵢ<ⱼ Aᵢⱼxᵢxⱼ

而在 F₂ 中：

xᵢ² = xᵢ
2 = 0

因此上式化简为：

0 = ∑ᵢ xᵢ = bᵀx

这就说明 b 与 Ker Aᵀ 中所有向量都正交，所以 b ∈ Im A。

---

### 2.3 结论

因此可以得到：

**定理**
对于任意有限无向图，如果一次操作会翻转某个点自身及其所有邻点，那么总可以通过有限次操作，把任意初始状态变到目标状态。

特别地，点灯问题一定有解。

这结论很硬，不会因为棋盘大小不同而失效。灯是跑不掉的，最多只是“解不唯一”。

---

## 3. 解的个数

既然解一定存在，下一步自然就是问：到底有多少个解？

一般线性方程组的答案是：

若 A 的秩为 r，那么解空间维数为 N−r，解的个数为 2ᴺ⁻ʳ

对于 n×n 棋盘，N=n²。
但如果直接把它当成 n² 个未知数去做高斯消元，复杂度会非常大。于是我们要利用棋盘的特殊结构，把问题压缩成一维递推。

---

## 4. 棋盘上的递推化简

现在考虑 m×n 棋盘。

把第 i 列的点击状态记为列向量 Xᵢ ∈ F₂ᵐ。
那么按完所有按钮以后，第 i 列受到的总影响来自四部分：

1. 左边一列的点击 Xᵢ₋₁
2. 右边一列的点击 Xᵢ₊₁
3. 本列上下相邻与自身造成的影响
4. 边界之外视为 0

若记 B 为 m×m 的三对角矩阵，其主对角线和上下副对角线全为 1，那么齐次方程可写成：

Xᵢ₋₁ + B Xᵢ + Xᵢ₊₁ = 0，i=1,2,…,n

并带边界条件：

X₀ = 0，Xₙ₊₁ = 0

这说明，一旦第一列 X₁ 确定，后面所有列都会被递推唯一决定。

---

## 5. 统一成项目里的多项式记号

原始文献里常用的是一组 pₖ(x) 递推。为了和你项目统一，我们改写成下面这组 Fibonacci 型多项式：

f₀(x)=1
f₁(x)=x
fₙ(x)=x·fₙ₋₁(x)+fₙ₋₂(x)，n≥2

则列向量递推可以写成：

Xᵢ = fᵢ₋₁(B) X₁，i=1,2,…,n+1

特别地，边界条件 Xₙ₊₁=0 变成：

fₙ(B) X₁ = 0

于是，整个 m×n 棋盘的解空间维数，就等于线性变换 fₙ(B) 的核空间维数：

d = dim Ker fₙ(B)

也就是说，问题已经从“求整张棋盘的解空间”，压缩成了“求一个多项式作用在 B 上以后，它的核有多大”。

这就是数学优化的第一刀：
二维棋盘 → 一维递推 → 多项式算子。

---

## 6. 为什么会出现 cₙ(x)=fₙ(x+1)

接下来是关键一步。

对于 m×m 的那块三对角矩阵 B，它的特征多项式恰好满足与 fₙ 类似的递推。进一步可以证明，它正好就是：

cₘ(x) = fₘ(x+1)

也就是说，在你的项目记号里：

cₘ(x) := fₘ(x+1)

所以矩阵 B 满足它自己的特征多项式：

cₘ(B+I) = fₘ((B+I)+1) = fₘ(B) = 0

更直接地说：
对高度为 m 的棋盘，矩阵 B 的特征多项式就是 fₘ(x+1)。

这一步非常值钱，因为从这里开始，问题里出现的两个核心多项式就统一成了：

fₙ(x)
cₘ(x)=fₘ(x+1)

而当 m=n 时，就正是你项目里一直在研究的那对：

fₙ(x)，fₙ(x+1)

---

## 7. 核空间维数为什么等于 gcd 的次数

现在设：

g(x) = gcd(fₙ(x), cₘ(x)) = gcd(fₙ(x), fₘ(x+1))

那么可以证明：

dim Ker fₙ(B) = deg g(x)

因此，m×n 棋盘的解空间维数就是：

d = deg gcd(fₙ(x), fₘ(x+1))

特别地，当棋盘是方形，即 m=n 时，有：

d = deg gcd(fₙ(x), fₙ(x+1))

也就是：

d = deg gₙ(x)

其中

gₙ(x) = gcd(fₙ(x), fₙ(x+1))

于是解的个数就是：

2ᵈ = 2^(deg gₙ)

这正是你项目里的核心结论。

---

## 8. 这一步背后的线性代数含义

这件事的本质是：

1. B 的极小多项式和特征多项式相同
2. 因而 B 在某个循环基下是“一个单块”的
3. 对于任意整除特征多项式的多项式 d(x)，都有：

dim Ker d(B) = deg d

所以只要把 d(x) 取成：

d(x)=gcd(fₙ(x), fₘ(x+1))

就立刻得到：

dim Ker fₙ(B)=deg d(x)

这是为什么“解空间维数”最后能变成“某个 gcd 的次数”的根本原因。

说白了就是：
你本来在数一个矩阵核空间的维数，结果由于这个矩阵太规整了，最后这件事完全等价于数一个多项式 gcd 的次数。

矩阵问题，硬是被打扁成多项式问题了。

---

## 9. 方形棋盘的最终公式

对于 n×n 点灯问题，解空间维数为：

dₙ = deg gcd(fₙ(x), fₙ(x+1))

记

gₙ(x) = gcd(fₙ(x), fₙ(x+1))

则

dₙ = deg gₙ(x)

所以解的个数为：

2^(deg gₙ)

这就是你项目里常写的版本：

解的个数 = 2^deg(gₙ)

其中

gₙ(x)=gcd(fₙ(x), fₙ(x+1))

---

## 10. 一个直接推论：什么时候唯一解

若 n = 2ʳ−1，则 n×n 点灯问题有唯一解。

原因是这时可以证明：

fₙ(x) = xⁿ

于是

fₙ(x+1) = (x+1)ⁿ

显然 xⁿ 与 (x+1)ⁿ 互素，因此：

gₙ(x)=gcd(xⁿ,(x+1)ⁿ)=1

从而：

deg gₙ = 0

所以解空间维数为 0，解只有一个。

也就是说：

n=1,3,7,15,31,…
这些大小的方形棋盘，解都是唯一的。

这类尺寸像开了挂一样，根本没有自由度。

---

## 11. 总结版

把整件事压缩成最核心的三句话，就是：

### 1. 递推多项式

f₀(x)=1
f₁(x)=x
fₙ(x)=x·fₙ₋₁(x)+fₙ₋₂(x)

### 2. 平移多项式

cₙ(x)=fₙ(x+1)

### 3. 解空间维数

gₙ(x)=gcd(fₙ(x),cₙ(x))=gcd(fₙ(x),fₙ(x+1))

dₙ = deg gₙ(x)

解的个数 = 2^dₙ = 2^(deg gₙ)

---

## 12. 一句人话总结

点灯问题之所以有意思，是因为：

1. 它表面上是一个棋盘翻灯游戏
2. 本质上是 F₂ 上的线性方程组
3. 再往下挖，会化成 Fibonacci 型多项式
4. 最终解的个数，居然只由 gcd(fₙ(x), fₙ(x+1)) 的次数决定

这就很离谱，也很优雅：
一堆灯泡最后被一个 gcd 管住了。

> 你要证明 `b` 能做出来，不如去检查：所有“做不出效果的隐藏方向”，会不会卡住 `b`。

## 3. 为什么“核空间维数 = gcd 的次数”

这一条最重要，我分两刀讲。原文其实也是这么干的，只是写得比较硬。

---

### 第 3.1 步：为什么先会冒出 gcd

原文先得到条件：

`fₙ(B)X₁ = 0`

但它不想直接算 `Ker fₙ(B)`，因为那样还是太硬。
于是它观察到：这个 `B` 本身满足自己的特征多项式，而在你项目记法里，这个特征多项式就是：

`cₘ(x)=fₘ(x+1)`

也就是你项目里一直用的那句：

`c(x)=f(x+1)`。

所以对矩阵 `B` 来说，有：

`cₘ(B)=0`

于是现在矩阵同时满足两件事：

* `fₙ(B)X₁=0`
* `cₘ(B)=0`

那么真正起作用的，不是 `fₙ` 本身，也不是 `cₘ` 本身，而是它们的**公共部分**。
这就是为什么原文定义：

`d(x)=gcd(fₙ(x), cₘ(x))`

在方阵 `m=n` 时就是：

`gₙ(x)=gcd(fₙ(x), fₙ(x+1))`

也就是你项目里现在的主角。

---

### 这一步的人话

把它想成两把锁：

* 一把锁是 `fₙ(B)`
* 一把锁是 `cₘ(B)`

而矩阵 `B` 自己已经被第二把锁锁死了：`cₘ(B)=0`。
所以第一把锁里真正额外有意义的部分，不是全部，而只是它和第二把锁**共同拥有的那截齿**。这截齿就是 `gcd`。

所以原文才会说：

> `X₁` 是解，当且仅当 `d(B)X₁=0`

也就是从 “看 `fₙ(B)` 的核” 变成了 “看 `d(B)` 的核”。

---

### 第 3.2 步：为什么 `dim Ker d(B)=deg d`

这一步原文用的是“不变因子/循环向量”的语言。
翻成人话，其实是在说：

> 这个三对角矩阵 `B` 很规整，规整到整个空间都可以由一个起始向量 `v` 不断乘 `B` 生成出来。

也就是：

`v, Bv, B²v, ..., Bᵐ⁻¹v`

这 m 个东西已经张成整个空间。

于是任意向量 `w` 都能唯一写成：

`w = p(B)v`

其中 `p(x)` 的次数小于 `m`。

现在要求 `w ∈ Ker d(B)`，就是要求：

`d(B)w = 0`

代进去得：

`d(B)p(B)v = 0`

由于 `B` 的最小多项式就是它的特征多项式 `cₘ(x)`，所以这件事等价于：

`cₘ(x) | d(x)p(x)`

现在关键来了：

* `d(x)` 已经是 `cₘ(x)` 的一个因子
* 所以 `p(x)` 只要补上“剩下那部分”就行
* 能自由选择的系数个数，刚好就是 `deg d`

于是就得到：

`dim Ker d(B)=deg d`

---

### 最通俗的一句话

这一步其实是在说：

> 因为 `B` 太规整了，所以“被 d(B) 干掉的向量有多少维”，完全等价于“多项式 d(x) 有多少阶”。

也就是：

矩阵核空间维数
= 多项式公共因子的次数

这一步很像魔术，但骨架是：

1. 先把任意向量写成 `p(B)v`
2. 再把“属于核空间”变成“整除条件”
3. 最后数自由系数个数

---

### 这条和你项目的直接对应

在你的项目里，方阵时就是：

* `cₙ(x)=fₙ(x+1)`
* `gₙ(x)=gcd(fₙ(x), cₙ(x))`

所以：

`dim Ker fₙ(B) = deg gₙ(x)`

这就是你后来一直写的：

`解的个数 = 2^deg(gₙ)`

的根。不是拍脑袋来的，是这条证明挖出来的。

`dₙ = deg gcd(fₙ(x), fₙ(x+1))`

也就是：

`dₙ = deg gₙ(x)`

---

### 为什么解的个数是 2^deg(gₙ)

因为我们整个问题都在 `F₂` 上。

如果解空间维数是 `dₙ`，那每个自由方向只有两种选法：取 0 或取 1。
所以总解数就是：

`2^{dₙ} = 2^{deg(gₙ)}`

---

### 最通俗的一句话

第 4 个证明其实没新技术，它只是把前面的成果装配起来：

> 第一列有多少自由度
> = 某个核空间有多少维
> = 某个 gcd 有多少阶

于是：

> 解有多少个
> = 2^(这个 gcd 的次数)

这就是整个“解的个数”章节真正落地的地方。

---

## 5. 为什么 n = 2ʳ−1 时唯一解

这一条原文是走分裂域闭式。
我给你一个更适合你项目风格、而且更直观的讲法：**用 doubling 公式秒掉它**。

你项目里已经有这两条 doubling 公式：

* `f₂ₘ(x)=fₘ(x)² + fₘ₋₁(x)²`
* `f₂ₘ₋₁(x)=x·fₘ₋₁(x)²`

---

### 先看几个例子

* `f₁(x)=x`
* `f₃(x)=x³`
* `f₇(x)=x⁷`

你会发现：

`f_{2ʳ−1}(x)=x^{2ʳ−1}`

---

### 为什么这是真的

用归纳法就行。

假设已经有：

`f_{2ʳ−1}(x)=x^{2ʳ−1}`

那么令 `m=2ʳ`，由第二条 doubling 公式：

`f_{2m-1}(x)=x·f_{m-1}(x)²`

而 `m-1=2ʳ−1`，所以：

`f_{2^{r+1}-1}(x)=x·(x^{2ʳ−1})² = x^{2^{r+1}-1}`

归纳完成。

---

### 那为什么这会导致唯一解

因为这时：

`fₙ(x)=xⁿ`
`cₙ(x)=fₙ(x+1)=(x+1)ⁿ`

所以

`gₙ(x)=gcd(xⁿ,(x+1)ⁿ)=1`

原因很简单：

* `x` 的根是 0
* `x+1` 的根是 1
* 它们没有公共根，所以互素

于是：

`deg gₙ(x)=0`

所以解空间维数为 0，解的个数就是：

`2⁰=1`

也就是唯一解。

---

### 最通俗的一句话

这条证明的本质是：

> 当 n=2ʳ−1 时，fₙ(x) 直接塌成纯幂 `xⁿ`，
> 而平移后就变成 `(x+1)ⁿ`，
> 这俩谁也不搭谁，所以 gcd 只能是 1。

于是自由度没了，只剩唯一解。

---

# 最后我帮你压缩成一句“存档版”

这 5 个证明连起来，其实就是这条主线：

1. 先证明点灯问题**总有解**
2. 再把二维棋盘压成**第一列递推**
3. 再把第一列的自由度，转成**核空间维数**
4. 再把核空间维数，转成**gcd 的次数**
5. 最后在特殊尺寸 `n=2ʳ−1` 下，算出 gcd=1，所以**唯一解**

用你项目记号收尾就是：

* `cₙ(x)=fₙ(x+1)`
* `gₙ(x)=gcd(fₙ(x),cₙ(x))`
* `dₙ=deg gₙ(x)`
* `解的个数 = 2^{deg gₙ(x)}`
